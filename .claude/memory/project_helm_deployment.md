## Cluster setup
- Runtime: `kind` (local-cluster), single-node
- Config: `cluster/kind-config.yaml`
- Registry: `cluster/helm-components.yaml` — single source of truth for all components
- Namespaces: `db` (postgres), `af` (airflow), `s3` (minio), `misc` (nginx + wiremock)

## Helm component registry (`cluster/helm-components.yaml`)

| Component | Chart | Namespace | Enabled | Depends on |
|-----------|-------|-----------|---------|------------|
| postgres | bitnami/postgresql | db | **false** | — |
| airflow | apache-airflow/airflow | af | true | postgres |
| minio | minio/minio | s3 | **false** | — |
| wiremock | wiremock/wiremock | misc | true | — |
| nginx | bitnami/nginx | misc | true | airflow, wiremock |

**Why postgres and minio are disabled:** The K8s stack uses only airflow + wiremock + nginx by default. Postgres runs as a bitnami chart (separate release in `db` namespace) because airflow needs it as a dependency — but the `postgres` entry in the registry is disabled to avoid double-deploying. MinIO is disabled because the K8s stack doesn't include a full query stack (iceberg-rest/trino not in helm registry yet).

## Secrets architecture
Created by `make kube-secrets` → `scripts/kube-secrets.sh` before any Helm deploy.

| Secret | Namespace | Keys | Used by |
|--------|-----------|------|---------|
| postgres-credentials | db | `password`, `postgres-password` | bitnami/postgresql `auth.existingSecret` |
| airflow-metadata | af | `connection` (SQLAlchemy URI) | airflow chart `data.metadataSecretName` |
| airflow-webserver | af | `webserver-secret-key` | airflow chart `webserverSecretKeySecretName` + extraEnv |
| airflow-fernet | af | `fernet-key` | airflow chart `fernetKeySecretName` |

ConfigMaps:
- `misc/wiremock-stubs` — built from `compose/wiremock/mappings/` via `--from-file`

All secrets sourced from `.env.local` (generated by `make secrets`). **Never add secrets to values.yaml.**

## Airflow Helm specifics (Airflow 3.2.0)
- Executor: LocalExecutor (no Celery/Redis/workers)
- DAGs: PVC `airflow-dags-pvc` (pre_manifests: `dags_pv.yaml` + `dags_pvc.yaml`)
- `migrateDatabaseJob.useHelmHooks: false` — avoids Helm hook race with slow postgres startup
- `dagProcessor.enabled: true` — required for Airflow 3 dag-parsing separation
- Triggerer probe: timeoutSeconds=60, periodSeconds=90 — tuned for slow `airflow jobs check` on kind

## Port-forwards (via `make kube-port-forward`)
| Service | Local | Remote |
|---------|-------|--------|
| postgres-postgresql (db) | 5432 | 5432 |
| airflow-api-server (af) | 8080 | 8080 |
| nginx (misc) | 8080 | 80 — conflicts with airflow! |
| wiremock (misc) | 8090 | 80 |

⚠ **Nginx and airflow-api-server both forward to localhost:8080** — only one can be active at a time.

## Nginx Helm proxy routes (localhost:8080 after port-forward)
| Path | Backend |
|------|---------|
| /airflow/ | airflow-api-server.af:8080 |
| /minio/ | minio-console.s3:9001 (MinIO disabled — 502) |
| /minio-api/ | minio.s3:9000 (MinIO disabled — 502) |
| /mock/ | wiremock.misc:80 |
| /health | nginx stub (200 OK) |

## Known gaps / TODOs
- `helm/minio/values.yaml` has hardcoded `rootPassword: "minio123"` (issue #3)
- No securityContext on any pod (issue #4/#21)
- No NetworkPolicy resources (issue #22/#5)
- No PodDisruptionBudget (issue #15)
- No namespace ResourceQuota (issue #26)
- `kube-secrets.sh` doesn't create `s3` namespace or MinIO secrets — enabling MinIO requires manual steps (GAP-K3)
- Nginx routes to disabled MinIO cause 502 on /minio/ and /minio-api/ (GAP-K2)
- Nginx and airflow-api-server port-forward conflict on :8080 (GAP-K-PF)
- Airflow logs not persisted (`logs.persistence.enabled: false`); ephemeral per pod restart (GAP-K4)

**Why:** Use this to understand the K8s deployment order, what make targets to run, and what's not yet safe for production use.
**How to apply:** When adding a new Helm component, register in `cluster/helm-components.yaml`, add values file under `helm/<name>/values.yaml`, add any required secrets to `scripts/kube-secrets.sh`.
